# tool_namespace 默认 collaboration 与 5.6 后端保留工具名冲突，hide=false 时全 turn 失败

Status: needs-triage

帖 6 楼（引 openai/codex#31864）：`tool_namespace` 默认 `"collaboration"` 时客户端
提交 `collaboration.spawn_agent`，GPT-5.6 后端把该名当保留工具、要求参数 schema
与服务端预设完全一致；一旦 `hide_spawn_agent_metadata = false` 让本地 schema 多出
`model` / `reasoning_effort` / `agent_type` 字段，schema 不匹配导致 Sol 所有 turn
失败。楼主给出的解法是 `tool_namespace = "agents"`。

对本项目的影响：显式路由要求 hide=false（见票 01），恰好落入该冲突组合——
即本项目的运行前提组合是「hide=false **且** tool_namespace="agents"」，二者缺一不可。

现有覆盖情况：validator 的 tool_name allowlist 已含 `agentsspawn_agent` /
`collaborationspawn_agent` 变体（src/codex_subagent_router/validator.py:13-20），
但那只覆盖 hook 侧识别；#31864 的失败发生在模型请求层，hook 名单救不了。
0.144.3 研究已确认 hook 侧 tool_name 归一化为 `spawn_agent`
（.scratch/release-readiness/research/v1-v2-spawn-contract.md:8）。

待办：

1. 核实 #31864 的影响版本范围与本项目支持矩阵
   （.scratch/release-readiness/issues/08-support-matrix.md）的交集。
2. 若在支持范围内，preflight 校验 `tool_namespace = "agents"`，与票 01 的
   hide 前提在同一处校验，修复面同样待用户定。
