# 安装器写入的 [agents.<role>] 表与显式启用 features.multi_agent_v2 的共存性未验证

Status: ready-for-agent

本项目安装器把 managed role 声明写进 config.toml 的 `[agents.<role>]` 表
（plan 侧检查见 src/codex_subagent_router/installation.py:198-221，receipt
一致性见 src/codex_subagent_router/_installation_files.py:195-200）。

帖 37 楼实测：config 中 V1 标量键与显式 V2 块共存时 Codex 直接报错
`agents.max_threads cannot be set when features.multi_agent_v2 is enabled`。
被拒的是 `agents` 命名空间下的标量键；`[agents.<role>]` 子表是否同样被拒未知。
另外 V2 世界的子代理模板官方机制是 `~/.codex/agents/<name>.toml` 文件
（支持 model / model_reasoning_effort / sandbox_mode / developer_instructions，
见 https://developers.openai.com/codex/subagents ），不是 config.toml 内的表。

现有探针（docs/research/codex-0.144.3-hook-evidence.md）在默认配置下跑通了
managed role 合同注入，但 config 里没有显式 `[features.multi_agent_v2]` 块——
用户若按帖子的流行配置显式启用 V2，安装产物的行为未被覆盖。

待办（探针任务，方法沿用 0.144.3 隔离探针先例）：

1. 隔离 codex home：config 同时含安装器写入的 `[agents.<role>]` 表与
   `[features.multi_agent_v2] enabled = true`，验证 Codex 是否启动报错、
   managed role 是否仍被发现、SubagentStart 合同注入是否仍生效。
2. 若冲突：评估 role 声明迁移到 `agents/*.toml` 机制；迁移时顺带评估把
   四个只读 role 用 `sandbox_mode = "read-only"` 从提示词软约束升级为硬约束
   （帖 47/114 楼讨论，楼主态度为可加可不加）。
